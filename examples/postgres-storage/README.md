# Postgres Storage effect-auth Example

The minimal auth flow backed by the built-in Drizzle Postgres storage adapter.

This example keeps database setup explicit:

1. Generate Drizzle schema TypeScript with `effect-auth generate`.
2. Let Drizzle Kit generate and apply SQL migrations.
3. Commit the generated `drizzle/` migrations as app-owned schema history.
4. Run the same sign-up, email verification, sign-in, and current-session flow as `examples/minimal`.

## Run with Docker

Run from the repository root:

```bash
docker compose -f examples/postgres-storage/docker-compose.yml down -v

docker compose -f examples/postgres-storage/docker-compose.yml up -d --wait
bun run build
bun run --cwd examples/postgres-storage auth:schema
bun run --cwd examples/postgres-storage db:generate
bun run --cwd examples/postgres-storage db:migrate
bun run example:postgres
```

`down -v` resets the example database so repeated runs start from a blank migration history. `--wait` is important: Drizzle Kit should not run migrations until Postgres passes the compose healthcheck.

The example defaults to:

```text
postgres://effect_auth:effect_auth@localhost:5432/effect_auth_example
```

Override it with `DATABASE_URL` for both migration and runtime commands:

```bash
DATABASE_URL="postgres://user:password@localhost:5432/my_auth_db" bun run --cwd examples/postgres-storage db:migrate
DATABASE_URL="postgres://user:password@localhost:5432/my_auth_db" bun run --cwd examples/postgres-storage demo
```

## What to read

- `src/schema.ts` is generated by `effect-auth generate` and exports top-level Drizzle table values for Drizzle Kit, including `OAuthStates` for storage-backed OAuth State.
- `drizzle.config.ts` points Drizzle Kit at `src/schema.ts` and the local Postgres URL.
- `drizzle/` is generated by Drizzle Kit and committed as this example app's migration history.
- `src/database.ts` wires `PgClient.layer` and `DrizzlePg.layer`; it does not create or mutate tables at runtime.
- `src/main.ts` composes Postgres storage with `MockAuthEmail`, an explicit `BoundedDevRateLimiter`, and runs the auth flow.

For production apps, run `npx effect-auth generate`, then `npx drizzle-kit generate`, then `npx drizzle-kit migrate`. Provide real Auth Storage, Auth Email, and Rate Limiter layers to `AuthLive(config?)`.
