# Postgres Storage effect-auth Example

Runs the same kind of auth flow as `examples/minimal`, but persists every auth record through the built-in Drizzle Postgres `AuthStorage` adapter.

## Flow

1. Generate the Drizzle auth schema TypeScript file with `effect-auth generate`.
2. Let Drizzle Kit generate and apply SQL migrations from that schema.
3. Create an Effect SQL Postgres client with `PgClient.layer(...)`.
4. Provide `DrizzlePg.layer({ schema })` as `AuthStorage`.
5. Bridge `effect-email/test` to `AuthEmail` for inspectable no-network emails.
6. Run an end-to-end workflow: sign-up, email verification, two sign-ins, session listing, password change, current-session lookup, and user deletion.

The password-change step demonstrates storage-owned atomic invariants: the current session token is rotated and other sessions are revoked in the same storage operation. The delete-user step shows the adapter deleting dependent auth records.

## Run with Docker

Run from the repository root:

```bash
docker compose -f examples/postgres-storage/docker-compose.yml up -d --wait
bun run build
bun run --cwd examples/postgres-storage auth:schema
bun run --cwd examples/postgres-storage db:generate
bun run --cwd examples/postgres-storage db:migrate
bun run example:postgres
```

`--wait` is important: Drizzle Kit should not run migrations until Postgres passes the compose healthcheck.

The example defaults to:

```text
postgres://effect_auth:effect_auth@localhost:5432/effect_auth_example
```

Override it with `DATABASE_URL` for both migration and runtime commands:

```bash
DATABASE_URL="postgres://user:password@localhost:5432/my_auth_db" bun run --cwd examples/postgres-storage db:migrate
DATABASE_URL="postgres://user:password@localhost:5432/my_auth_db" bun run --cwd examples/postgres-storage demo
```

## Important Files

- `src/schema.ts` is generated by `effect-auth generate` and exports top-level Drizzle table values for Drizzle Kit.
- `drizzle.config.ts` points Drizzle Kit at `src/schema.ts` and the local Postgres URL.
- `src/database.ts` wires `PgClient.layer` and `DrizzlePg.layer`; it does not create or mutate tables at runtime.
- `src/email.ts` bridges `effect-email/test` to `AuthEmail`.
- `src/main.ts` composes the layers and runs the full storage-backed auth flow.

For production apps, run `npx effect-auth generate`, then `npx drizzle-kit generate`, then `npx drizzle-kit migrate`. Provide a real email adapter plus `AuthLive.production` with a production `RateLimiter`.
