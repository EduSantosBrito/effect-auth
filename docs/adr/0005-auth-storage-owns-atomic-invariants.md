# Auth Storage Owns Atomic Invariants

Effect Auth keeps multi-step auth invariants inside **Auth Storage** instead of exposing only primitive persistence calls to core workflows or end-user code. Packaged **Storage Adapters** implement behavioral operations such as password reset completion, password change, user deletion, session revocation, session rotation, and OAuth sign-in/linking transactionally, because partial commits across token consumption, password updates, deleted identities, sessions, provider accounts, and provider token updates would weaken the security model. OAuth storage behavior owns uniqueness and race-sensitive invariants such as one Provider Account per External Provider subject, unique Normalized Emails, automatic same-email linking, idempotent same-user linking, and safe failure when a Provider Account is already linked to another User.
